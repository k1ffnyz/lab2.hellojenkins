## Теория

### CI/CD

Непрерывная интеграция (CI, Continuous Integration) — это практика разработки программного обеспечения, при которой изменения в коде регулярно автоматически собираются, тестируются и проверяются. Основная цель CI — как можно раньше обнаруживать ошибки интеграции и снижать стоимость их исправления.

Непрерывное развертывание (CD, Continuous Deployment) дополняет CI и позволяет автоматически доставлять и разворачивать приложение в целевой среде после успешного прохождения всех этапов сборки и тестирования.

Для реализации CI/CD используются специализированные инструменты, такие как Jenkins, GitLab CI, GitHub Actions, TeamCity и другие.

---

### Jenkins

Jenkins — это система автоматизации сборки и развертывания, предназначенная для реализации процессов CI/CD. Jenkins обладает модульной архитектурой и поддерживает расширение функциональности с помощью плагинов.

Основные преимущества Jenkins:
- открытый исходный код;
- широкая экосистема плагинов;
- поддержка Pipeline как кода;
- интеграция с Docker, GitHub и другими инструментами разработки.

Jenkins функционирует по модели контроллер–агенты, где контроллер управляет заданиями, а агенты выполняют сборки.
